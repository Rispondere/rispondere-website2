## 1. 管理画面へのアクセス方法

### ステップ1: URLにアクセス

ブラウザのアドレスバーに以下のURLを入力します：

```
https://YOUR-SITE.com/admin/
```

**注意**: 最後の `/` (スラッシュ) を忘れずに入力してください。

### ステップ2: ページが表示される

正しくアクセスできると、以下のような画面が表示されます：


## 2. ログイン方法

### 方法A: GitHub認証（推奨）

#### ステップ1: 「GitHubでログイン」ボタンをクリック

管理画面に表示される「GitHubでログイン」ボタンをクリックします。

#### ステップ2: GitHub認証画面

GitHubのログイン画面が表示されます。
- GitHubアカウントの **ユーザー名/メールアドレス** を入力
- **パスワード** を入力
- 「Sign in」をクリック

#### ステップ3: 権限の承認

初回のみ、アプリケーションからの権限要求が表示されます。
- 「Authorize」（承認）ボタンをクリック

#### ステップ4: ログイン完了

管理画面のダッシュボードが表示されます。

---

### 方法B: Netlify Identity（代替方法）

**注意**: この方法を使用する場合は、事前にNetlify Identityの設定が必要です。

#### ステップ1: メールアドレスを入力

管理画面に表示されるメールアドレス入力欄に、登録済みのメールアドレスを入力します。

#### ステップ2: パスワードを入力

設定したパスワードを入力します。

#### ステップ3: ログイン

「ログイン」ボタンをクリックします。

## 4. デザインカラーの変更

サイト全体の配色を変更できます。

### ステップ1: デザイン設定を開く

1. 左サイドバーの **「デザイン設定」** をクリック
2. **「デザインカラー」** をクリック

### ステップ2: カラーを編集

以下の4つのカラーを設定できます：

```
┌─────────────────────────────────────┐
│  デザインカラー編集                  │
├─────────────────────────────────────┤
│                                     │
│  メインカラー                        │
│  ┌──────┐  #d32f2f                 │
│  │ 🔴   │  ← カラーピッカー        │
│  └──────┘                           │
│                                     │
│  サブカラー                          │
│  ┌──────┐  #f44336                 │
│  │ 🔴   │                           │
│  └──────┘                           │
│                                     │
│  背景色                              │
│  ┌──────┐  #ffffff                 │
│  │ ⬜   │                           │
│  └──────┘                           │
│                                     │
│  文字色                              │
│  ┌──────┐  #333333                 │
│  │ ⬛   │                           │
│  └──────┘                           │
│                                     │
│  [保存]  [キャンセル]               │
│                                     │
└─────────────────────────────────────┘
```

#### 各カラーの役割

- **メインカラー**: ボタン、見出し、リンクなど主要な色
- **サブカラー**: ホバー時、グラデーションの補助色
- **背景色**: ページ全体の背景
- **文字色**: 本文テキストの色

### ステップ3: カラーの選択方法

#### 方法1: カラーコードを直接入力
```
#d32f2f ← 6桁の16進数カラーコード
```

#### 方法2: カラーピッカーを使用
1. カラーボックス（色のついた四角）をクリック
2. カラーピッカーが開く
3. 好きな色をクリックまたはドラッグして選択

### ステップ4: 保存

1. 右上の **「保存」** ボタンをクリック
2. 確認メッセージが表示される
3. サイトに反映される（数分かかる場合があります）

- **表示順** の数字を変更すると、FAQページでの表示順序が変わります
- 小さい数字ほど上に表示されます（1が最上位）

## 9. 画像のアップロード

サイトで使用する画像をアップロードできます。

### ステップ1: メディアライブラリを開く

1. 管理画面上部の **「メディア」** アイコンをクリック
2. または、画像選択時に **「新しい画像をアップロード」** を選択

### ステップ2: 画像をアップロード

#### 方法1: ドラッグ&ドロップ

1. パソコンの画像ファイルを選択
2. メディアライブラリの画面にドラッグ
3. 自動的にアップロードされる

#### 方法2: ファイルを選択

1. **「ファイルを選択」** ボタンをクリック
2. パソコンのファイル選択ダイアログが開く
3. 画像ファイルを選択
4. 「開く」をクリック

### ステップ3: 画像の推奨サイズと容量

| 用途 | 推奨サイズ（px） | アスペクト比 | 推奨容量 | 最大容量 | 形式 | 備考 |
|------|----------------|-------------|---------|---------|------|------|
| **ヒーローバナー（TOP横長）** | 1920×600 | 16:5 | 200〜500KB | 1MB | JPG/PNG | レスポンシブ対応、横幅1920px推奨 |
| **ヒーローバナー（小）** | 1200×400 | 3:1 | 100〜300KB | 1MB | JPG/PNG | 軽量版、モバイル対応 |
| **縦長バナー（採用ページ）** | 400×800 | 1:2 | 100〜300KB | 800KB | JPG/PNG | 縦長デザイン、スマホ表示に最適 |
| **縦長バナー（標準）** | 300×600 | 1:2 | 50〜200KB | 500KB | JPG/PNG | サイドバー・採用ページ用 |
| **実績ギャラリー** | 800×600 | 4:3 | 100〜300KB | 800KB | JPG/PNG | グリッド表示、ホバーエフェクト |
| **実績ギャラリー（小）** | 600×400 | 3:2 | 50〜150KB | 500KB | JPG/PNG | サムネイル用 |
| **ロゴ（ヘッダー用）** | 200×50 | 4:1 | 10〜50KB | 100KB | PNG/SVG | 透過PNG推奨、SVG最適 |
| **ロゴ（高解像度）** | 400×100 | 4:1 | 20〜100KB | 200KB | PNG/SVG | Retina対応 |
| **OGP画像（SNS共有）** | 1200×630 | 1.91:1 | 200〜500KB | 1MB | JPG/PNG | Facebook/Twitter推奨サイズ |
| **ファビコン** | 512×512 | 1:1 | 10〜50KB | 100KB | PNG | 正方形、シンプルなデザイン |

#### 📏 サイズ選びのポイント

1. **横幅は最大1920px**：大画面モニターでも綺麗に表示
2. **高さは600〜800px**：画面全体を占めすぎない
3. **縦長バナーは400×800px**：スマホでの視認性が高い
4. **ロゴはSVG形式が最適**：拡大縮小しても綺麗

#### 💾 容量最適化のコツ

1. **写真（JPG）**：
   - 推奨: 200〜500KB
   - 最大: 1MB
   - 圧縮ツール: [TinyJPG](https://tinyjpg.com/)、[Squoosh](https://squoosh.app/)

2. **イラスト・ロゴ（PNG）**：
   - 推奨: 50〜200KB
   - 最大: 500KB
   - 透過が必要な場合のみPNG、それ以外はJPG

3. **ロゴ（SVG）**：
   - 推奨: 5〜20KB
   - 最大: 50KB
   - 拡大縮小しても綺麗、容量も最小

#### 🎨 画像形式の使い分け

| 形式 | 用途 | メリット | デメリット |
|------|------|---------|-----------|
| **JPG** | 写真、風景 | 容量が小さい | 透過不可 |
| **PNG** | ロゴ、透過画像 | 透過対応、高品質 | 容量が大きい |
| **SVG** | ロゴ、アイコン | 超軽量、拡大縮小自由 | 複雑な画像には不向き |
| **WebP** | 次世代形式 | JPGより30%軽量 | 古いブラウザ非対応 |

#### 🔧 画像圧縮ツール（無料）

1. **TinyPNG/TinyJPG** - https://tinypng.com/
   - PNG/JPGを最大70%圧縮
   - 品質劣化が少ない
   - 使い方: ドラッグ&ドロップ → ダウンロード

2. **Squoosh** - https://squoosh.app/
   - Google製の画像圧縮ツール
   - リアルタイムプレビュー
   - WebP/AVIF形式にも対応

3. **ImageOptim**（Mac専用） - https://imageoptim.com/
   - ドラッグ&ドロップで自動圧縮
   - 複数ファイル一括処理

4. **JPEG-Optimizer** - http://jpeg-optimizer.com/
   - オンライン圧縮ツール
   - 圧縮率を細かく設定可能

#### ⚠️ アップロード時の注意点

1. **ファイル名は英数字のみ**：
   - ❌ `バナー画像.jpg`
   - ✅ `banner-image.jpg`

2. **容量制限**：
   - GitHub: 1ファイル100MB（推奨: 1MB以下）
   - Netlify: 1ファイル無制限（ただし転送速度に影響）

3. **ファイル形式**：
   - 対応: JPG、PNG、SVG、GIF、WebP
   - 非対応: PSD、AI、TIFF、BMP

## 11. トラブルシューティング

### Q1. ログインできない

**症状**: 「ログイン」ボタンを押しても何も起きない

**原因と対処法**:

1. **GitHubアカウントの権限不足**
   - リポジトリへのアクセス権限があるか確認
   - リポジトリ管理者に権限を付与してもらう

2. **ブラウザのキャッシュ問題**
   - Ctrl+Shift+Delete（Windows）または Cmd+Shift+Delete（Mac）
   - キャッシュをクリア
   - ブラウザを再起動

3. **ポップアップブロック**
   - ブラウザのポップアップブロックを無効化
   - GitHubのドメインを許可リストに追加

---

### Q2. 画像がアップロードできない

**症状**: 画像を選択しても「アップロード失敗」と表示される

**原因と対処法**:

1. **ファイルサイズが大きすぎる**
   - 画像を圧縮（推奨: 1MB以下）
   - TinyPNG (https://tinypng.com/) などで圧縮

2. **対応していないファイル形式**
   - JPG、PNG、SVG形式を使用
   - GIF、WEBP は使用できない場合あり

3. **ファイル名に日本語**
   - ファイル名を英数字に変更
   - 例: `バナー画像.jpg` → `banner-image.jpg`

---

### Q3. 保存ボタンが押せない（グレーアウト）

**症状**: 保存ボタンがクリックできない

**原因と対処法**:

1. **必須項目が未入力**
   - 赤い枠や「必須」マークがある項目を確認
   - すべて入力する

2. **変更がない**
   - 何も編集していない場合は保存できない
   - 何か変更を加える

---

### Q4. 保存したのにサイトに反映されない

**症状**: 「公開」ボタンを押したが、サイトが更新されない

**原因と対処法**:

1. **反映に時間がかかる**
   - GitHub Pages: 1〜3分待つ
   - Netlify: 30秒〜1分待つ

2. **ブラウザキャッシュ**
   - Ctrl+F5（Windows）または Cmd+Shift+R（Mac）
   - ハードリフレッシュを実行

3. **ビルドエラー**
   - GitHubのリポジトリ → Actions タブを確認
   - エラーがあれば管理者に連絡

---

### Q5. 編集中の内容が消えた

**症状**: 編集中にページを閉じてしまい、内容が消えた

**対処法**:

1. **自動保存機能**
   - Decap CMSは自動保存しない
   - こまめに「保存」ボタンをクリック

2. **下書きを確認**
   - 左サイドバーの「下書き」を確認
   - 一時保存されている場合がある

3. **予防策**
   - 長文は外部エディタ（メモ帳など）で作成
   - コピー&ペーストで貼り付け

## 🎓 画像アップロードの完全ガイド

### ステップ1: 画像を準備

#### 1-1. 画像のリサイズ（サイズ変更）

**無料ツール**:
- **Canva** (https://www.canva.com/) - ブラウザで使える
- **GIMP** (https://www.gimp.org/) - 無料のPhotoshop代替
- **Photopea** (https://www.photopea.com/) - オンライン版Photoshop

**手順**（Canvaの場合）:
1. Canvaにアクセス
2. 「カスタムサイズ」で横幅1920px、高さ600pxを指定
3. 画像をアップロード
4. 配置・調整
5. ダウンロード（JPG形式、品質80〜90%）

#### 1-2. 画像の圧縮（容量削減）

**TinyPNGの使い方**:
1. https://tinypng.com/ にアクセス
2. 画像をドラッグ&ドロップ（最大20枚まで一括処理）
3. 圧縮完了まで待つ（数秒）
4. 「Download」ボタンで保存

**圧縮前後の比較例**:
```
圧縮前: banner.jpg - 2.5MB
圧縮後: banner.jpg - 450KB (82%削減!)
```

#### 1-3. ファイル名の変更

**良い例**:
- ✅ `hero-banner.jpg`
- ✅ `recruit-banner-01.jpg`
- ✅ `gallery-work-01.jpg`
- ✅ `logo-rispondere.svg`

**悪い例**:
- ❌ `バナー画像.jpg` （日本語）
- ❌ `IMG_1234.jpg` （意味不明）
- ❌ `作業実績　最新版.png` （スペース・日本語）

#### 方法1: メディアライブラリから

2. 「Upload」ボタンをクリック
3. ファイルを選択、またはドラッグ&ドロップ
4. アップロード完了を確認

#### 方法2: 編集画面から直接

1. バナー設定などで「画像を選択」をクリック
2. 「新しい画像をアップロード」を選択
3. ファイルを選択
4. 自動的にアップロードされ、選択される

### ステップ3: 画像を確認

アップロード後、以下を確認：
- ✅ ファイル名が正しい
- ✅ サイズが適切（容量が1MB以下）

## 🚨 画像に関するトラブルシューティング

### Q7. 画像がぼやける・荒い

**原因**:
- 元画像が小さすぎる
- 圧縮しすぎた

**対処法**:
1. より大きいサイズの元画像を使用（推奨: 横幅1920px以上）
2. 圧縮率を下げる（TinyPNGで70〜80%に調整）
3. PNG形式で保存（品質優先）

### Q8. 画像がサイトに表示されない

**原因と対処法**:

1. **ファイル名に日本語**
   - 英数字に変更して再アップロード

2. **容量が大きすぎる**
   - TinyPNGで圧縮（目標: 500KB以下）

3. **キャッシュの問題**
   - ブラウザをハードリフレッシュ（Ctrl+F5）
   - 1〜2分待って再度確認

### Q9. 画像の透過が効いていない

**原因**:
- JPG形式で保存している（JPGは透過非対応）

**対処法**:
1. PNG形式で保存し直す
2. 透過部分を確認
3. 再度アップロード

### Q10. アップロードが遅い

**原因**:
- 画像容量が大きすぎる

**対処法**:
1. 画像を圧縮（目標: 500KB以下）
2. 複数ファイルを一度にアップロードしない
3. Wi-Fi環境で作業
