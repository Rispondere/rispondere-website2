## 🚀 次のステップ（デプロイ手順）

### 1. GitHubにプッシュ

```bash
cd /home/user/webapp

# リモートリポジトリを追加（まだの場合）
git remote add origin https://github.com/[USERNAME]/[REPO_NAME].git

# プッシュ
git push -u origin main
```

### 2. GitHub Pages有効化

1. GitHubリポジトリのSettings → Pages
2. Source: `Deploy from a branch`
3. Branch: `main` / Folder: `/ (root)`
4. Save
5. 数分後に公開URL取得

### 3. Decap CMS設定更新

`admin/config.yml` を編集：

```yaml
backend:
  name: github
  repo: [USERNAME]/[REPO_NAME]  # ← 実際のリポジトリ名に変更
  branch: main
```

コミット＆プッシュ：

```bash
git add admin/config.yml
git commit -m "Update: CMS repository configuration"
git push
```

### 4. Netlify認証設定（管理画面用）

1. Netlifyアカウント作成（無料）
2. New site from Git → GitHub連携
3. Deploy（何も設定せず）
4. Site settings → Identity → Enable Git Gateway
5. 完了！`/admin/` からログイン可能

## 📊 実装した勝ち要素の詳細

### ① 安心設計（未経験でも大丈夫が根拠付き）

✅ **実装箇所**:
- 仕事内容説明に「未経験からスタートした人も在籍」明記
- よくある質問に「未経験でも大丈夫ですか?」を設置
- 業務は段階的に覚える旨を明示
- 「まずは話だけ聞きたい」も歓迎と明記

### ② 具体的な仕事の景色（1日の流れ・成果物・関わる人）

✅ **実装箇所**:
- **1日の流れ**: 10:00〜19:00のタイムライン形式
- **具体的な業務**: カード形式で6つの業務を説明
- **関わる人**: 問い合わせ対応・チーム全体のサポート等を明示
- **成果物**: 制作進行・データ整理・Web更新等を具体化

### ③ 向いている人／向かない人（ミスマッチ削減）

✅ **実装箇所**:
- **向いている人**: 4項目をリスト化
  - コツコツと丁寧に進めることが得意な人
  - 複数のことを整理しながら進められる人
  - 人とのやりとりが苦にならない人
  - 長期的に腰を据えて働きたい人
- **向かない人**: 3項目をリスト化
  - 短期的な仕事を探している人
  - 制作メインで働きたい人
  - 日々変わらない環境で働きたい人

### ④ 選考フローが明確（電話→説明→面談→次）

✅ **実装箇所**:
- **STEP 1**: 電話・メールで問い合わせ
- **STEP 2**: 仕事内容の説明
- **STEP 3**: 面談
- **STEP 4**: 条件確認・採用
