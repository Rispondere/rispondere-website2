## 🎯 このマニュアルでできること

このマニュアルを読めば、以下の作業ができるようになります：

- ✅ **サイトの画像を変更する**（ロゴ、制作実績の画像など）
- ✅ **サイトの文章を変更する**（トップページの見出し、説明文など）
- ✅ **スプレッドシートでサイト全体を管理する**（一括更新）

**所要時間**: 各作業 5〜10分程度

## 📸 画像の変更方法

### ステップ1: 画像を準備する

#### 推奨サイズ

| 画像の種類 | 推奨サイズ | 形式 |
|----------|----------|-----|
| ロゴ | 横400px × 縦100px | PNG |
| 制作実績画像 | 横800px × 縦450px（16:9） | JPEG |
| その他の画像 | 横800px以上 | JPEG/PNG |

#### 画像の準備手順

1. **画像を用意する**
   - スマホで撮影した写真でもOK
   - パソコンに保存しておく

2. **画像のサイズを確認する**
   - Windowsの場合: 画像を右クリック → プロパティ → 詳細
   - Macの場合: 画像を選択 → Cmd+I

3. **ファイル名を分かりやすくする**
   - 例: `works_01_spa.jpg`（制作実績1枚目）
   - 例: `logo_new.png`（新しいロゴ）

1. **GitHubにアクセス**

## 📊 スプレッドシートの使い方

### スプレッドシートとは？

サイト全体の文章・画像URLを**一箇所で管理できる仕組み**です。

- ✅ **Excel感覚で編集できる**
- ✅ **サイト全体を一括更新できる**
- ✅ **間違えても元に戻せる**

---

### ステップ1: スプレッドシートを開く

1. **Google スプレッドシートにアクセス**

2. **Googleアカウントでログイン**
   - アカウント: （担当者に確認）

3. **「settings」シートを開く**
   ```
   下部のタブから「settings」をクリック
   ```

---

### ステップ2: スプレッドシートの見方

#### スプレッドシートの構造

| 列 | 説明 | 例 |
|----|-----|---|
| **A列（key）** | 項目の名前（変更しない） | hero_title |
| **B列（value）** | 実際の文章・画像URL（ここを変更） | デザインを、止めない。 |
| **C列（description）** | 説明（参考用） | トップページのメイン見出し |

#### 例: トップページの見出し

| key | value | description |
|-----|-------|------------|
| hero_title | デザインを、止めない。 | トップページのメイン見出し |

---

### ステップ3: スプレッドシートで変更する

#### 例: トップページの見出しを変更する

1. **「hero_title」の行を探す**
   - 検索: `Ctrl + F` → `hero_title` と入力

2. **B列（value）の内容を変更する**
   
   **変更前**:
   ```
   デザインを、止めない。
   ```

   ```

   ```

3. **自動保存される**
   - 左上に「すべての変更を保存しました」と表示される

✅ **スプレッドシートの変更完了！**

## ❓ よくある質問

### Q1. 変更が反映されない

**A1. 以下を確認してください**

1. **GitHub Pagesの反映待ち**
   - 変更後、2〜5分待ってから確認

2. **キャッシュの影響**
   - `Ctrl + Shift + R`（Windows）でキャッシュをクリア
   - `Cmd + Shift + R`（Mac）でキャッシュをクリア

3. **ファイル名の間違い**
   - 大文字・小文字、スペルを確認

4. **コミットの確認**
   - GitHubで「Commits」を確認して、変更がプッシュされているか確認

---

### Q2. 画像が表示されない

**A2. 以下を確認してください**

1. **画像のパスが正しいか確認**
   ```html
   <!-- 正しい例 -->
   <img src="/images/logo/works_01.jpg">
   
   <!-- 間違い例 -->
   <img src="images/logo/works_01.jpg">  ← 先頭の / がない
   <img src="/images/works_01.jpg">      ← logo/ がない
   ```

2. **画像ファイルがアップロードされているか確認**
   - GitHub の `images/logo/` フォルダを開く
   - ファイルが存在するか確認

3. **ファイル名が正しいか確認**
   - 大文字・小文字を確認（`Works_01.jpg` と `works_01.jpg` は別物）
   - スペースが入っていないか確認

4. **画像のサイズが大きすぎないか確認**
   - 推奨: 2MB以下
   - 大きすぎる場合は圧縮ツールを使用

---

### Q3. スプレッドシートの変更がサイトに反映されない

**A3. Apps Scriptを実行していますか？**

1. **スプレッドシートで変更しただけでは反映されません**
   - Apps Scriptを実行する必要があります

2. **実行方法**
   - メニュー「🔄 Rispondere更新」→「データを更新」をクリック
   - または Apps Script エディタから `updateRispondereSettings` を実行

3. **実行後、2〜3分待ってから確認**
